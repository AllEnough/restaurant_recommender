# 簡報大綱：今天吃什麼：餐廳與食譜智慧推薦系統

## 第 1 頁：封面

標題：今天吃什麼：餐廳與食譜智慧推薦系統

副標題：結合外食與內食情境之智慧飲食推薦系統

建議內容：

- 課程名稱
- 組員姓名與學號
- 指導老師
- 報告日期

講解重點：

本專題希望解決大學生常見的「不知道吃什麼」問題，並將使用者分成外食族與內食族兩種情境。

---

## 第 2 頁：專題動機

建議標題：為什麼需要這個系統？

建議內容：

- 外食族常需要比較價格、距離、評分與餐點類型。
- 內食族常因為冰箱食材有限，不知道可以煮什麼。
- 現有平台資訊很多，但使用者仍需要自己整理與判斷。
- 本系統希望降低使用者搜尋與決策成本。

講解重點：

我們不是只做單純查詢，而是讓系統根據使用者條件計算推薦分數，並提供推薦理由。

---

## 第 3 頁：系統目標

建議標題：專題目標

建議內容：

1. 建立餐廳與食譜資料集。
2. 使用 Python 進行資料處理與分析。
3. 設計外食餐廳加權推薦模型。
4. 設計內食食譜食材比對模型。
5. 加入情緒策略推薦。
6. 使用互動式地圖呈現推薦餐廳。
7. 使用 Streamlit 建立可線上展示的系統。

講解重點：

系統同時支援「外食推薦」與「內食推薦」，讓不同使用情境都能被涵蓋。

---

## 第 4 頁：系統架構

建議標題：系統架構圖

建議內容：

```text
使用者
  ↓
Streamlit 網站介面
  ↓
選擇使用情境
  ├─ 外食推薦
  │   ├─ restaurants.csv
  │   ├─ 餐廳推薦模型
  │   ├─ 情緒策略加權
  │   └─ 地圖與圖表
  │
  └─ 內食推薦
      ├─ recipes.csv
      ├─ 食材比對模型
      └─ 食譜推薦結果
```

講解重點：

整個系統以 CSV 作為資料來源，透過 Python 模型計算推薦結果，再由 Streamlit 顯示互動式介面。

## 第 5 頁：外食推薦功能

建議標題：外食族餐廳推薦

建議內容：

使用者可輸入：

- 預算上限
- 可接受距離
- 餐點類型
- 目前天氣
- 目前心情
- 外帶需求
- 可接受辣度
- 是否希望快速出餐

輸出結果：

- 推薦餐廳排名
- 推薦分數
- 推薦理由
- 推薦分數圖表

講解重點：

外食推薦不是隨機挑選，而是綜合多個條件進行加權評分。

---

## 第 6 頁：外食推薦模型

建議標題：餐廳推薦分數計算

建議內容：

```text
餐廳推薦分數 =
評分分數
+ 價格符合度
+ 距離符合度
+ 類型符合度
+ 天氣與心情符合度
+ 外帶符合度
+ 辣度符合度
+ 出餐速度分數
+ 情緒策略加權
```

補充說明：

- 分數範圍限制在 0 到 100。
- 分數越高代表越符合使用者需求。
- 系統會同時產生推薦理由，提高可解釋性。

講解重點：

這一頁要強調技術含量：系統有設計推薦模型，而不是簡單 if-else 查詢。

---

## 第 7 頁：情緒策略推薦

建議標題：心情如何影響推薦？

建議內容：

| 心情 | 推薦策略 |
|---|---|
| 省錢 | 低價與 CP 值高的餐廳加分 |
| 疲累 | 距離近與快速出餐餐廳加分 |
| 開心 | 高評分與適合聚餐餐廳加分 |
| 心情不好 | 甜點、飲料、炸物、小吃加分 |
| 選擇困難 | 從高分餐廳中產生驚喜推薦 |

講解重點：

心情不只是顯示文字，而是會改變推薦分數與排序。

---

## 第 8 頁：虛擬美食地圖

建議標題：互動式餐廳地圖

建議內容：

- 使用 folium 與 streamlit-folium 製作互動式地圖。
- 推薦餐廳會以圖針標示位置。
- 第一名推薦餐廳使用紅色圖針。
- 點擊圖針可以看到餐廳資訊、推薦分數、CP 值與推薦理由。

建議放圖：

- 外食推薦地圖截圖

講解重點：

地圖讓推薦結果具備空間感，使用者可以更直覺地理解餐廳位置。

## 第 9 頁：內食食譜推薦功能

建議標題：內食族食譜推薦

建議內容：

使用者輸入：

- 冰箱現有食材
- 可接受烹飪時間
- 料理難度

系統輸出：

- 推薦食譜排名
- 推薦分數
- 符合食材
- 缺少食材
- 烹飪時間
- 難度與熱量

講解重點：

內食推薦解決的是「有食材但不知道煮什麼」的問題。

---

## 第 10 頁：內食推薦模型

建議標題：食譜推薦分數計算

建議內容：

```text
食譜推薦分數 =
食材符合度
+ 烹飪時間符合度
+ 難度符合度
+ 基礎分數
- 缺少食材懲罰
```

補充說明：

- 使用者食材越接近食譜所需食材，分數越高。
- 缺少食材太多時，分數會下降。
- 系統會列出符合食材與缺少食材，方便使用者判斷可行性。

講解重點：

這一頁要說明內食推薦的技術核心是食材集合比對與分數計算。
