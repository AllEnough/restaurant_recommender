# 簡報大綱：智慧飲食決策系統

## 第 1 頁：封面

標題：智慧飲食決策系統

副標題：結合評論情緒分析與食材保存優先級之飲食決策輔助系統

建議內容：

- 課程名稱
- 組員姓名與學號
- 指導老師
- 報告日期

講解重點：

本專題希望解決大學生常見的飲食決策問題，並將使用者分成外食族與內食族兩種情境：外食降低踩雷風險，內食降低食材浪費。

---

## 第 2 頁：研究動機

建議標題：為什麼需要這個系統？

建議內容：

- 外食族常需要比較價格、距離、評分與餐點類型。
- 內食族常因為冰箱食材有限，不知道可以煮什麼。
- 食材放太久容易過期，造成浪費。
- 現有平台資訊很多，但使用者仍需要自己整理與判斷。
- 本系統希望降低使用者搜尋成本、決策成本、外食踩雷風險與內食食材浪費。

講解重點：

我們不是只做單純查詢，而是讓系統根據使用者條件計算推薦分數，並提供推薦理由。

---

## 第 3 頁：專題目標

建議標題：專題目標

建議內容：

1. 建立餐廳與食譜資料集。
2. 使用 Python 進行資料處理與推薦計算。
3. 設計外食餐廳加權推薦模型。
4. 加入餐廳評論文字分析。
5. 加入推薦模型解釋器。
6. 加入權重敏感度分析。
7. 加入決策分析 Dashboard。
8. 加入模型評估，比較模型調整前後的推薦變化。
9. 加入自動天氣判斷，讓系統依照即時環境調整推薦。
10. 加入自動用餐時段判斷，讓系統依照使用時間調整推薦。
11. 加入瀏覽器定位，讓系統依照使用者實際位置重新計算距離。
12. 設計內食食譜食材比對模型。
13. 加入情緒策略推薦。
14. 加入食材保存優先級排序。
15. 使用互動式地圖呈現推薦餐廳。
16. 使用 Streamlit 建立可線上展示的系統。

講解重點：

系統同時支援「外食決策」與「內食決策」，並將外食推薦升級成能考慮評論風險，將內食推薦升級成能考慮食材保存狀態的決策系統。

## 第 5 頁：功能一：自動情境與情緒推薦

建議標題：系統如何判斷現在適合吃什麼？

建議內容：

智慧模式：

| 模式 | 目的 |
|---|---|
| 快速午餐 | 近距離、快速出餐、可外帶 |
| 省錢外食 | 預算較低、CP 值優先 |
| 不想踩雷 | 高評分、低負評比例 |
| 疲累近距離 | 距離與速度優先 |

自動情境判斷：

| 情境 | 判斷方式 | 影響 |
|---|---|---|
| 天氣 | 取得即時天氣資料 | 普通、熱、冷、雨天 |
| 用餐時段 | 根據目前時間判斷 | 早餐、午餐、下午茶、晚餐、宵夜 |
| 心情 | 使用者選擇 | 調整餐點類型與排序 |

心情策略：

| 心情 | 推薦策略 |
|---|---|
| 省錢 | 低價與 CP 值高的餐廳加分 |
| 疲累 | 距離近與快速出餐餐廳加分 |
| 開心 | 高評分與適合聚餐餐廳加分 |
| 心情不好 | 甜點、飲料、炸物、小吃加分 |
| 選擇困難 | 從高分餐廳中產生驚喜推薦 |

建議放圖：

- 外食模式畫面截圖
- 自動天氣與時段判斷截圖
- 心情選單截圖
- 推薦理由截圖

講解重點：

系統不是只靠使用者手動輸入，而是先用智慧模式降低操作成本，再自動取得天氣、判斷用餐時段，並結合使用者心情調整推薦分數與排序。這讓外食決策更貼近使用者當下情境。

## 第 6 頁：功能二：冰箱食材推薦

建議標題：從冰箱食材推回適合食譜

建議內容：

使用者輸入：

- 內食智慧模式
- 冰箱現有食材
- 可接受烹飪時間
- 料理難度
- 熱量上限
- 最多可缺少食材數

系統輸出：

- 本次最佳建議摘要
- 推薦食譜排名
- 推薦分數
- 符合食材
- 缺少食材
- 食材優先加權
- 缺少食材購物清單

建議放圖：

- 內食推薦畫面
- 食譜推薦卡片
- 缺少食材購物清單

講解重點：

內食決策解決的是「有食材但不知道煮什麼」的問題。使用者可以選擇我不想出門、清冰箱模式、快速料理或低熱量，系統會自動調整推薦條件，並讓使用者知道還缺哪些材料。

## 第 7 頁：功能三：虛擬美食地圖

建議標題：互動式餐廳地圖與 CP 值標記

建議內容：

- 使用 folium 與 streamlit-folium 製作互動式地圖。
- 推薦餐廳會以圖針標示位置。
- 紅色圖針代表本次推薦第一名。
- 綠色圖針代表高 CP 值餐廳。
- 橘色圖針代表中高 CP 值餐廳。
- 點擊圖針可以看到餐廳資訊、推薦分數、CP 值與推薦理由。

建議放圖：

- 外食推薦地圖截圖

講解重點：

地圖讓推薦結果具備空間感，圖針顏色則讓使用者更快看出哪間餐廳 CP 值較高。

## 第 10 頁：推薦模型設計

建議標題：推薦分數如何計算？

建議內容：

外食推薦分數：

```text
評分分數
+ 價格符合度
+ 距離符合度
+ 類型符合度
+ 天氣與心情符合度
+ 外帶符合度
+ 辣度符合度
+ 出餐速度分數
+ 情緒策略加權
+ 用餐時段加權
```

內食推薦分數：

```text
食材符合度
+ 烹飪時間符合度
+ 難度符合度
+ 熱量符合度
- 缺少食材懲罰
+ 食材優先級加權
```

講解重點：

這一頁要強調技術含量：系統有設計推薦模型，而不是簡單 if-else 查詢。

## 第 11 頁：餐廳評論文字分析

建議標題：如何從評論看出口碑風險？

建議內容：

- 使用 `reviews.csv` 模擬餐廳評論資料。
- 透過正向與負向關鍵字計算情緒分數。
- 統計每間餐廳的負評比例。
- 擷取常見優點與常見疑慮。
- 將評論調整分數加入外食推薦排序。

講解重點：

這一頁可以強調系統加入文字探勘概念，不只是看餐廳評分，也會分析評論內容中的風險訊號。

## 第 12 頁：推薦模型解釋器

建議標題：為什麼系統推薦這間？

建議內容：

- 拆解餐廳推薦分數來源。
- 顯示評分、價格、距離、類型、情境、外帶、辣度、出餐速度、心情策略與評論調整。
- 使用長條圖呈現各項分數貢獻。
- 使用敏感度分析比較不同偏好下的第一名是否改變。

講解重點：

這一頁可以強調推薦系統的可解釋性。系統不只給出答案，也能說明答案是如何被計算出來的。

## 第 13 頁：食材優先級排序

建議標題：如何降低食材浪費？

建議內容：

使用者可設定：

- 已放天數
- 保存期限
- 估計價格
- 易腐程度

系統會計算：

- 優先分數
- 剩餘天數
- 排序比值
- 優先級

補充說明：

排序概念參考鞋匠排程問題：浪費成本越高、剩餘時間越短，越應該優先處理。

講解重點：

這是目前專題最能拉高技術含量的部分，因為它讓食譜推薦從「食材比對」升級成「食材使用順序決策」。
