# 智慧飲食決策系統

## 一、專題名稱

正式名稱：結合評論情緒分析與食材保存優先級之智慧飲食決策系統

展示名稱：智慧飲食決策系統

## 二、專題動機

大學生在日常生活中常遇到飲食決策困難。外食時，除了價格、距離與餐點類型，也需要判斷餐廳評論是否穩定，避免踩雷；內食時，則常因為冰箱食材有限，或食材快過期，不知道應該先煮什麼。

一般查詢平台雖然提供大量餐廳或食譜資訊，但使用者仍需要自行整理、比較與判斷。因此，本專題希望建立一套以 Python 開發的智慧飲食決策系統，將使用情境分成外食與內食兩種模式。外食族可透過條件輸入與評論分析取得餐廳建議，內食族可輸入現有食材與保存狀態取得食譜建議，降低搜尋成本、踩雷風險與食材浪費。

## 三、專題目標

1. 建立餐廳資料集與食譜資料集。
2. 使用 Python 與 pandas 進行資料讀取、整理與推薦計算。
3. 設計外食餐廳多條件加權推薦模型。
4. 加入餐廳評論文字分析，計算情緒分數、負評比例與常見關鍵字。
5. 加入推薦模型解釋器，拆解推薦分數來源。
6. 加入權重敏感度分析，比較不同偏好下的排序變化。
7. 加入決策分析 Dashboard，整理外食評論風險、推薦分數變化、內食食材浪費風險與食譜適配度。
8. 加入模型評估，觀察加入評論分析或保存優先級前後的推薦排名差異。
9. 加入自動天氣判斷，將即時天氣轉換成外食推薦條件。
10. 加入自動用餐時段判斷，依照早餐、午餐、下午茶、晚餐與宵夜調整推薦。
11. 加入瀏覽器定位，依照使用者實際位置重新計算餐廳步行距離。
12. 加入外食與內食智慧模式，將常見需求轉換成預設推薦條件。
13. 加入本次最佳建議摘要，讓使用者第一眼就能看懂系統建議。
14. 加入決策對比，說明第一名適合族群、關鍵取捨與第二名差異。
15. 加入條件過嚴時的自動放寬建議，提升系統可用性。
16. 加入回饋式偏好學習，讓喜歡或不喜歡影響後續推薦排序。
17. 設計內食食譜食材比對推薦模型。
18. 加入情緒策略，使心情實際影響餐廳推薦排序。
19. 加入食材保存優先級排序，使內食推薦能優先處理快過期或浪費成本較高的食材。
20. 提供推薦分數、推薦理由與推薦重點，提升推薦結果可解釋性。
21. 使用 Streamlit 建立可互動、可部署的網站介面。
22. 使用互動式地圖呈現推薦餐廳位置與 CP 值差異。

## 六、功能模組

### 1. 外食餐廳推薦模組

使用者可以輸入以下條件：

- 智慧模式，例如快速午餐、省錢外食、不想踩雷、疲累近距離
- 預算上限
- 可接受距離
- 餐點類型
- 目前天氣（自動偵測或手動選擇）
- 目前心情
- 是否需要外帶
- 可接受辣度
- 是否希望快速出餐
- 排序方式
- 最低評分

系統會根據上述條件計算每間餐廳的推薦分數，並輸出推薦前幾名餐廳。

外食模式支援自動天氣判斷。使用者輸入所在地區後，系統會透過公開天氣資料取得目前溫度與天氣描述，並轉換成推薦模型使用的普通、熱、冷、雨天四種分類。若天氣資料取得失敗，系統會自動使用普通天氣，也可改用手動選擇。

系統也支援自動用餐時段判斷。系統會依照目前時間推論早餐、午餐、下午茶、晚餐或宵夜，並提高對應餐點類型的推薦分數。使用者也可以改為手動選擇時段。

外食智慧模式會把常見使用情境轉換成一組合理的預設條件：

| 智慧模式 | 預設決策方向 |
|---|---|
| 快速午餐 | 近距離、快速出餐、可外帶、降低踩雷風險 |
| 省錢外食 | 預算較低、CP 值優先 |
| 不想踩雷 | 高評分、低負評比例、隱藏高風險餐廳 |
| 疲累近距離 | 距離最近、快速出餐、辣度較低 |

外食結果會進一步產生決策對比，包含適合族群、關鍵取捨與第一名相較第二名的優勢。若使用者條件太嚴格而沒有結果，系統會提供放寬建議，例如調整預算、距離、最低評分或評論風險門檻。

系統也加入回饋式偏好學習。使用者對餐廳按喜歡或不喜歡後，系統會根據餐點類型、價格區間、出餐速度、外帶需求、辣度與評論風險建立偏好權重，並調整後續推薦結果的最終分數。

### 2. 自動情境判斷模組

系統會自動判斷兩種外部情境，並轉換成推薦模型條件：

| 自動判斷項目 | 判斷方式 | 影響 |
|---|---|---|
| 天氣環境 | 使用 wttr.in JSON 取得溫度與天氣描述 | 轉換為普通、熱、冷、雨天 |
| 用餐時段 | 使用目前時間進行規則分類 | 轉換為早餐、午餐、下午茶、晚餐、宵夜 |
| 使用者位置 | 使用瀏覽器定位取得經緯度 | 重新計算餐廳步行距離 |

用餐時段加權規則如下：

| 用餐時段 | 加權餐點類型 |
|---|---|
| 早餐 | 早餐、早午餐、飯類、飲料 |
| 午餐 | 便當、飯類、麵食、小吃、水餃、鍋貼、素食 |
| 下午茶 | 飲料、甜點、輕食、早午餐 |
| 晚餐 | 火鍋、麵食、飯類、日式、韓式、義式、泰式、鐵板燒、美式、港式 |
| 宵夜 | 小吃、炸物、飲料、麵食、鍋貼、水餃 |

若自動判斷失敗或使用者想自行控制條件，系統也保留手動選擇。

使用者位置會透過 Haversine 公式計算與各餐廳的直線距離，再換算成預估步行分鐘。此距離會影響距離篩選、距離最近排序、疲累模式加權與地圖中心點。

### 3. 情緒策略推薦模組

系統會根據使用者心情調整推薦策略：

| 心情 | 推薦策略 |
|---|---|
| 省錢 | 低價與 CP 值高的餐廳加分 |
| 疲累 | 距離近與快速出餐的餐廳加分 |
| 開心 | 高評分與適合聚餐的餐廳加分 |
| 心情不好 | 甜點、飲料、炸物、小吃類餐廳加分 |
| 選擇困難 | 從高分餐廳中產生今日驚喜推薦 |

此功能讓心情不只是篩選條件，而是會實際影響推薦模型的分數。

### 4. 虛擬美食地圖模組

系統使用 folium 建立互動式地圖，將推薦餐廳以圖針方式標示在地圖上。為了讓地圖不只是位置顯示，系統會依照推薦排名與 CP 值改變圖針顏色：

| 圖針顏色 | 代表意義 |
|---|---|
| 紅色 | 本次推薦第一名 |
| 綠色 | 高 CP 值餐廳 |
| 橘色 | 中高 CP 值餐廳 |
| 藍色 | 一般推薦餐廳 |

使用者點擊圖針後，可以查看餐廳名稱、餐點類型、平均價格、評分、距離、推薦分數、CP 值與推薦理由。

### 5. 餐廳評論文字分析模組

系統使用 `reviews.csv` 模擬餐廳評論資料，並透過關鍵字方式分析評論內容。分析結果包含：

- 評論數量
- 評論情緒分數
- 負評比例
- 常見優點
- 常見疑慮
- 常見主題
- 評論摘要

外食決策可選擇是否納入評論分析。若啟用，系統會依照評論調整分數修正原本的推薦分數，使推薦結果能同時考慮條件符合度與評論風險。

### 6. 決策分析 Dashboard 模組

為了讓系統不只是輸出推薦清單，本專題加入決策分析 Dashboard：

- 外食 Dashboard：顯示候選數、最佳選項、平均負評比例、評論低風險餐廳數。
- 外食 Dashboard：比較原始分數與加入評論分析後的最終分數，觀察評論對排序的影響。
- 外食 Dashboard：統計各餐點類型的平均負評比例與平均評論情緒。
- 內食 Dashboard：顯示目前食材數、可直接料理數、平均缺少食材數與高優先食材估計價值。
- 內食 Dashboard：比較推薦食譜的符合食材數、缺少食材數、料理時間與保存加權。
- 內食 Dashboard：統計食譜類型的資料分布、平均時間與平均熱量。

此模組能在報告展示中呈現系統的資料分析能力，讓推薦結果更容易被理解與驗證。

### 7. 模型評估模組

本專題加入模型評估區塊，用來比較模型調整前後的推薦變化：

- 外食模型評估：比較未納入評論分析與納入評論分析後的 Top 推薦重疊率。
- 外食模型評估：顯示第一名是否改變、平均負評比例是否下降、平均評論情緒是否提升。
- 外食模型評估：列出各餐廳原始排名、評論後排名、原始分數、最終分數與評論調整。
- 內食模型評估：比較未納入保存優先級與納入保存優先級後的 Top 推薦重疊率。
- 內食模型評估：顯示高優先食材使用次數是否提升，以及平均缺少食材數是否改變。
- 內食模型評估：列出各食譜原始排名、保存加權後排名、保存加權與優先食材。

此模組可以用來說明模型設計是否真的影響推薦結果，也讓報告展示更接近資料科學專題。

### 8. 推薦模型解釋器與敏感度分析模組

外食決策結果中，系統提供推薦模型解釋器，讓使用者選擇任一餐廳後，查看該餐廳的分數來源：

- 評分分數
- 價格符合度
- 距離符合度
- 餐點類型符合度
- 天氣與心情情境分數
- 外帶、辣度、出餐速度分數
- 心情策略加權
- 評論調整分數

此外，系統提供權重敏感度分析，將推薦結果用不同情境重新比較，例如目前排序、省錢優先、距離優先、評分優先、口碑優先與低負評優先。若不同情境下第一名改變，代表推薦結果對使用者偏好較敏感；若第一名一致，代表推薦結果較穩定。

### 9. 內食食譜推薦模組

使用者可以輸入冰箱現有食材，例如：

```text
雞蛋, 白飯, 蔥
```

系統會比對 recipes.csv 中的食譜資料，計算食材符合度，並依照烹飪時間、料理難度、熱量限制與缺少食材數量產生食譜推薦。

內食智慧模式包含我不想出門、清冰箱模式、快速料理與低熱量。系統會依照模式自動調整缺少食材數、烹飪時間、熱量限制與是否只顯示可直接料理的食譜，讓使用者能更快取得可行結果。

內食結果也會產生決策對比，包含食譜適合族群、關鍵取捨與第一名相較第二名的差異。若沒有結果，系統會根據目前條件提示可放寬缺少食材數、料理時間、熱量上限或可料理限制。

內食模式也加入回饋式偏好學習。使用者對食譜按喜歡或不喜歡後，系統會根據食譜類型、料理難度、料理時間區間、熱量區間與食材特徵建立偏好權重，並影響後續排序。

推薦結果包含：

- 食譜名稱
- 食譜類型
- 烹飪時間
- 難度
- 熱量
- 符合食材
- 缺少食材
- 推薦分數
- 食材優先加權
- 優先食材
- 推薦理由

### 10. 食材保存優先級模組

為了讓內食決策更有技術含量，系統加入食材保存資訊：

- 已放天數
- 保存期限
- 估計價格
- 易腐程度

系統會計算每個食材的優先分數、剩餘天數、排序比值與優先級。排序概念參考鞋匠排程問題：浪費成本越高、剩餘時間越短的食材，越應該優先處理。

此模組會將食材優先級轉換成食譜推薦加權，讓推薦結果不只是判斷「能不能煮」，也能判斷「應該先煮什麼」。

### 11. 推薦摘要與收藏模組

系統會在推薦結果上方提供本次推薦重點，協助使用者快速判斷結果：

- 外食模式：顯示最高 CP 值餐廳、最近餐廳、最高評分餐廳。
- 內食模式：顯示最快完成食譜、食材最接近食譜、熱量最低食譜。
- 收藏功能：使用者可收藏喜歡的餐廳或食譜，並於側邊欄查看。
- 幫我決定：當使用者選擇困難時，系統可從目前推薦結果中直接選出一筆。
- 缺少食材購物清單：統計目前推薦食譜常缺少的食材。

## 八、推薦模型設計

### 1. 外食餐廳推薦分數

```text
餐廳推薦分數 =
評分分數
+ 價格符合度
+ 距離符合度
+ 類型符合度
+ 天氣與心情符合度
+ 外帶符合度
+ 辣度符合度
+ 出餐速度分數
+ 情緒策略加權
+ 用餐時段加權
```

系統會將分數限制在 0 到 100 分之間，分數越高代表越符合使用者需求。

### 2. 內食食譜推薦分數

```text
食譜基礎分數 =
食材符合度
+ 烹飪時間符合度
+ 難度符合度
+ 熱量符合度
- 缺少食材懲罰

食譜最終分數 =
食譜基礎分數
+ 食材優先級加權
```

當使用者輸入的食材越接近食譜所需食材，推薦分數越高；若食譜使用到高優先級食材，系統會額外加分。

### 3. 食材優先級計算概念

```text
食材優先級 =
保存期限使用比例
+ 即將過期加權
+ 價格加權
+ 易腐程度加權
```

```text
排序比值 =
浪費成本 / 剩餘天數
```

浪費成本會受到價格與易腐程度影響，剩餘天數越少，排序比值越高，代表越需要優先使用。

## 九、系統特色

1. 同時支援外食族與內食族兩種使用情境。
2. 使用加權評分模型，而不是單純隨機推薦。
3. 推薦結果具有可解釋性，能顯示推薦理由。
4. 心情會實際影響推薦分數，形成情緒化推薦策略。
5. 互動式地圖讓餐廳推薦更具空間感。
6. 評論文字分析讓系統能比較每間餐廳的口碑風險。
7. 模型解釋器讓推薦結果具備可解釋性。
8. 權重敏感度分析能說明不同使用者偏好如何影響排序。
9. 食材保存優先級讓內食決策更貼近真實生活需求。
10. 類鞋匠排程問題概念能增加專題的演算法敘事。
11. 使用推薦重點摘要、標籤、收藏與幫我決定功能，提升操作體驗。
12. 介面經過重新設計，較接近正式網站原型。
