# 今天吃什麼：餐廳與食譜智慧推薦系統

## 一、專題名稱

正式名稱：結合外食與內食情境之餐廳與食譜智慧推薦系統

展示名稱：今天吃什麼：餐廳與食譜智慧推薦系統

## 二、專題動機

大學生在日常生活中常遇到「不知道吃什麼」的問題。外食時，需要比較餐廳的價格、距離、評分、出餐速度與個人偏好；內食時，則常因為冰箱食材有限，不知道可以煮出什麼料理。

因此，本專題希望建立一套以 Python 開發的智慧飲食推薦系統，將使用情境分成外食與內食兩種模式。外食族可透過條件輸入取得餐廳推薦，內食族可輸入現有食材取得食譜推薦，降低使用者搜尋與決策成本。

## 三、專題目標

1. 建立餐廳資料集與食譜資料集。
2. 使用 Python 與 pandas 進行資料讀取、清理與分析。
3. 設計外食餐廳多條件加權推薦模型。
4. 設計內食食譜食材比對推薦模型。
5. 依照使用者心情調整餐廳推薦策略。
6. 提供推薦分數與推薦理由，提升推薦結果可解釋性。
7. 使用 Streamlit 建立互動式網站介面。
8. 使用互動式地圖呈現推薦餐廳位置。

## 五、系統架構

```text
使用者
  ↓
Streamlit 網站介面
  ↓
選擇使用情境
  ├─ 外食推薦模式
  │   ├─ 讀取 restaurants.csv
  │   ├─ 使用者輸入預算、距離、天氣、心情等條件
  │   ├─ 餐廳加權評分模型
  │   ├─ 情緒策略加權
  │   ├─ 推薦理由產生
  │   └─ 地圖與圖表顯示
  │
  └─ 內食推薦模式
      ├─ 讀取 recipes.csv
      ├─ 使用者輸入冰箱現有食材
      ├─ 食材符合度比對
      ├─ 食譜推薦分數計算
      └─ 推薦理由與圖表顯示
```

## 六、功能模組

### 1. 外食餐廳推薦模組

使用者可以輸入以下條件：

- 預算上限
- 可接受距離
- 餐點類型
- 目前天氣
- 目前心情
- 是否需要外帶
- 可接受辣度
- 是否希望快速出餐

系統會根據上述條件計算每間餐廳的推薦分數，並輸出推薦前幾名餐廳。

### 2. 情緒策略推薦模組

系統會根據使用者心情調整推薦策略：

| 心情 | 推薦策略 |
|---|---|
| 省錢 | 低價與 CP 值高的餐廳加分 |
| 疲累 | 距離近與快速出餐的餐廳加分 |
| 開心 | 高評分與適合聚餐的餐廳加分 |
| 心情不好 | 甜點、飲料、炸物、小吃類餐廳加分 |
| 選擇困難 | 從高分餐廳中產生今日驚喜推薦 |

此功能讓心情不只是篩選條件，而是會實際影響推薦模型的分數。

### 3. 虛擬美食地圖模組

系統使用 folium 建立互動式地圖，將推薦餐廳以圖針方式標示在地圖上。使用者點擊圖針後，可以查看：

- 餐廳名稱
- 餐點類型
- 平均價格
- 評分
- 距離
- 推薦分數
- CP 值
- 推薦理由

此模組讓餐廳推薦具備空間感，方便使用者了解推薦餐廳的位置分布。

### 4. 內食食譜推薦模組

使用者可以輸入冰箱現有食材，例如：

```text
雞蛋, 白飯, 蔥
```

系統會比對 recipes.csv 中的食譜資料，計算食材符合度，並依照烹飪時間、料理難度與缺少食材數量產生食譜推薦。

推薦結果包含：

- 食譜名稱
- 食譜類型
- 烹飪時間
- 難度
- 熱量
- 符合食材
- 缺少食材
- 推薦分數
- 推薦理由

### 5. 資料視覺化模組

系統會顯示以下圖表：

- 外食推薦分數比較圖
- 餐廳類型數量圖
- 各類餐廳平均價格圖
- 食譜推薦分數比較圖
- 食譜類型數量圖
- 各類食譜平均烹飪時間圖

## 七、資料集設計

## 八、推薦模型設計

### 1. 外食餐廳推薦分數

餐廳推薦分數由多個條件組成：

```text
餐廳推薦分數 =
評分分數
+ 價格符合度
+ 距離符合度
+ 類型符合度
+ 天氣與心情符合度
+ 外帶符合度
+ 辣度符合度
+ 出餐速度分數
+ 情緒策略加權
```

系統會將分數限制在 0 到 100 分之間，分數越高代表越符合使用者需求。

### 2. 內食食譜推薦分數

食譜推薦分數主要由食材符合度與料理條件組成：

```text
食譜推薦分數 =
食材符合度
+ 烹飪時間符合度
+ 難度符合度
+ 基礎分數
- 缺少食材懲罰
```

當使用者輸入的食材越接近食譜所需食材，推薦分數越高；若缺少過多食材，分數會下降。

## 九、系統特色

1. 同時支援外食族與內食族兩種使用情境。
2. 使用加權評分模型，而不是單純隨機推薦。
3. 推薦結果具有可解釋性，能顯示推薦理由。
4. 心情會實際影響推薦分數，形成情緒化推薦策略。
5. 互動式地圖讓餐廳推薦更具空間感。
6. 使用圖表呈現資料分析結果，提升報告完整度。
