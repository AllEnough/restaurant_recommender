# 智慧飲食決策系統

本專題是一套使用 Python 與 Streamlit 製作的智慧飲食決策系統，目標是協助大學生降低外食踩雷風險，並減少內食時的食材浪費。

系統分成兩種使用情境：

- **外食決策**：根據預算、距離、餐點類型、自動天氣、心情、評論情緒與負評比例推薦餐廳，降低踩雷風險。
- **內食決策**：根據冰箱現有食材、料理條件、缺少食材數與食材保存狀態推薦食譜，優先使用快過期或浪費成本較高的食材。

## 專題特色

- 支援外食族與內食族兩種情境。
- 使用多條件加權評分模型推薦餐廳。
- 使用評論文字分析計算餐廳情緒分數、負評比例與常見關鍵字。
- 使用食材集合比對模型推薦食譜。
- 支援自動天氣判斷，將即時天氣轉換成推薦模型條件。
- 支援自動用餐時段判斷，依照早餐、午餐、下午茶、晚餐與宵夜調整推薦。
- 支援瀏覽器定位，依照使用者實際位置重新計算餐廳步行距離與排序。
- 根據使用者心情調整餐廳推薦策略。
- 提供決策分析 Dashboard，整理評論風險、推薦分數變化、食材浪費風險與食譜適配度。
- 提供模型評估區塊，比較加入評論分析或保存優先級前後的推薦排名差異。
- 提供推薦模型解釋器，拆解每間餐廳的分數來源。
- 提供權重敏感度分析，比較不同偏好下推薦第一名是否改變。
- 加入食材保存期限、已放天數、價格與易腐程度，計算食材使用優先級。
- 內食推薦參考鞋匠排程問題概念，優先使用浪費成本高且剩餘時間短的食材。
- 提供推薦分數、推薦理由、推薦標籤與本次推薦重點，提升可解釋性。
- 使用互動式地圖標示推薦餐廳位置，並用圖針顏色呈現 CP 值差異。
- 支援收藏與「幫我決定」功能，提升實際操作便利性。
- 使用 Streamlit 自訂 CSS 重整介面，讓網站更接近正式產品原型。
- 可部署到 Streamlit Community Cloud 供組員或老師線上查看。

## 功能介紹

### 1. 外食決策

使用者可以輸入：

- 預算上限
- 可接受距離
- 餐點類型
- 目前天氣（可自動偵測或手動選擇）
- 用餐時段（可自動判斷或手動選擇）
- 目前位置（可使用瀏覽器定位或手動選擇據點）
- 目前心情
- 是否需要外帶
- 可接受辣度
- 是否希望快速出餐
- 排序方式與最低評分

系統會輸出：

- 推薦餐廳排名
- 推薦分數
- 推薦理由
- 推薦標籤
- 本次推薦重點摘要
- 自動天氣判斷結果
- 自動用餐時段判斷結果
- 使用者位置與重新計算後的步行距離
- 外食決策分析 Dashboard
- 外食模型評估
- 今日外食籤
- 餐廳評論情緒分析
- 推薦餐廳互動地圖
- 完整餐廳資料表

### 2. 情緒策略推薦

外食模式支援自動天氣判斷。使用者輸入所在地區後，系統會透過公開天氣資料取得目前溫度與天氣描述，並轉換成推薦模型使用的四種分類：普通、熱、冷、雨天。若天氣資料取得失敗，系統會自動使用普通天氣作為備援，也可以改成手動選擇。

系統也會依照目前時間自動判斷用餐時段，例如早餐、午餐、下午茶、晚餐與宵夜，並提高對應餐點類型的推薦分數。使用者也可以改為手動選擇時段。

外食模式也支援瀏覽器定位。使用者允許定位後，系統會取得目前經緯度，使用 Haversine 公式計算使用者到每間餐廳的距離，並換算成預估步行分鐘。若使用者不允許定位，系統也提供逢甲大學、逢甲夜市、文華路商圈等手動據點作為備援。

不同心情會影響推薦分數：

| 心情 | 推薦策略 |
|---|---|
| 省錢 | 低價與 CP 值高的餐廳加分 |
| 疲累 | 距離近與快速出餐的餐廳加分 |
| 開心 | 高評分與適合聚餐的餐廳加分 |
| 心情不好 | 甜點、飲料、炸物、小吃加分 |
| 選擇困難 | 從高分餐廳中產生今日驚喜推薦 |

### 3. 虛擬美食地圖

外食決策結果會以圖針標示在互動式地圖上。系統會依照推薦結果與 CP 值改變圖針顏色：

- 紅色：本次推薦第一名
- 綠色：高 CP 值餐廳
- 橘色：中高 CP 值餐廳
- 藍色：一般推薦餐廳

圖針資訊包含餐廳名稱、餐點類型、平均價格、評分、距離、推薦分數、CP 值與推薦理由。

### 4. 餐廳評論文字分析

系統使用 `reviews.csv` 模擬餐廳評論資料，並透過關鍵字方式進行情緒分析：

- 計算每間餐廳的評論情緒分數。
- 統計負評比例。
- 擷取常見優點與常見疑慮。
- 歸納評論常見主題，例如價格、速度、口味、環境與份量。
- 將評論調整分數加入外食推薦排序。

此功能讓外食決策不只依賴評分與價格，也能分析使用者評論中隱含的風險與優勢。

### 5. 決策分析 Dashboard

系統新增外食與內食兩種 Dashboard，讓推薦結果具備資料分析與決策說明：

- 外食 Dashboard：顯示決策候選數、最佳選項、平均負評比例、評論低風險餐廳數。
- 外食 Dashboard：比較原始推薦分數與加入評論分析後的最終分數。
- 外食 Dashboard：統計不同餐點類型的平均負評比例與評論情緒。
- 內食 Dashboard：顯示目前食材數、可直接料理數、平均缺少食材數與高優先食材估計價值。
- 內食 Dashboard：比較食譜的符合食材數、缺少食材數、料理時間與保存加權。
- 內食 Dashboard：統計不同食譜類型的數量、平均時間與平均熱量。

### 6. 模型評估

系統提供推薦前後比較，讓專題不只展示結果，也能說明模型調整造成的影響：

- 外食模型評估：比較未納入評論分析與納入評論分析後的 Top 推薦重疊率。
- 外食模型評估：顯示第一名是否改變、平均負評比例是否下降、平均評論情緒是否提升。
- 外食模型評估：列出每間餐廳的原始排名、評論後排名、原始分數、最終分數與評論調整。
- 內食模型評估：比較未納入保存優先級與納入保存優先級後的 Top 推薦重疊率。
- 內食模型評估：顯示高優先食材使用次數是否提升，以及平均缺少食材數是否改變。
- 內食模型評估：列出每道食譜的原始排名、保存加權後排名、保存加權與優先食材。

### 7. 推薦模型解釋器

外食模式提供推薦模型解釋器，可選擇任一推薦餐廳並查看：

- 評分貢獻
- 價格貢獻
- 距離貢獻
- 餐點類型貢獻
- 天氣與心情情境貢獻
- 外帶、辣度與出餐速度貢獻
- 心情策略加權
- 評論調整分數

系統也提供權重敏感度分析，用不同情境重新觀察推薦第一名，例如目前排序、省錢優先、距離優先、評分優先、口碑優先與低負評優先。這能幫助使用者理解推薦結果是否穩定，也能在報告中呈現推薦模型的可解釋性。

### 8. 內食食譜推薦

使用者可以輸入冰箱現有食材，例如：

```text
雞蛋, 白飯, 蔥
```

系統會根據食材符合度、烹飪時間、料理難度、熱量上限與缺少食材數量推薦食譜。

輸出結果包含：

- 食譜名稱
- 食譜類型
- 料理時間
- 難度
- 熱量
- 符合食材
- 缺少食材
- 推薦分數
- 食材優先加權
- 優先食材
- 推薦理由

### 9. 食材保存優先級排序

內食模式中，使用者可以為每個冰箱食材設定：

- 已放天數
- 保存期限
- 估計價格
- 易腐程度

系統會計算：

- 優先分數
- 剩餘天數
- 排序比值
- 優先級高、中、低

排序概念參考鞋匠排程問題：如果某個食材越快壞掉、價格越高、易腐程度越高，代表浪費成本越大，因此應該優先被使用。系統會將食材優先級轉換成食譜推薦加權，讓推薦結果不只是「能不能煮」，也會考慮「應該先煮什麼」。

## 推薦模型概念

### 外食餐廳推薦分數

```text
餐廳推薦分數 =
評分分數
+ 價格符合度
+ 距離符合度
+ 類型符合度
+ 天氣與心情符合度
+ 外帶符合度
+ 辣度符合度
+ 出餐速度分數
+ 情緒策略加權
+ 用餐時段加權
```

分數限制在 0 到 100 分之間，分數越高代表越符合使用者需求。

### 內食食譜推薦分數

```text
食譜基礎分數 =
食材符合度
+ 烹飪時間符合度
+ 難度符合度
+ 熱量符合度
- 缺少食材懲罰

食譜最終分數 =
食譜基礎分數
+ 食材優先級加權
```

當使用者輸入的食材越接近食譜所需食材，推薦分數越高。若食譜使用到高優先級食材，系統會額外加分，讓即將過期或浪費成本較高的食材優先被使用。

## 技術含量說明

本專題的技術核心不是單純查資料，而是把使用者條件轉換成可計算的推薦分數：

- 外食決策使用多條件加權模型，綜合評分、價格、距離、類型、天氣、心情、外帶、辣度與出餐速度。
- 自動天氣判斷會把即時天氣轉換成普通、熱、冷、雨天，並影響外食情境分數。
- 自動用餐時段判斷會依照目前時間推論早餐、午餐、下午茶、晚餐或宵夜，並影響外食推薦分數。
- 瀏覽器定位會重新計算使用者到餐廳的步行距離，使距離篩選、距離最近排序與疲累模式更貼近實際位置。
- 情緒策略會依照不同心情額外調整分數，例如省錢模式提高低價與高 CP 值餐廳，疲累模式提高近距離與快速出餐餐廳。
- 評論分析會將評論文字轉換成情緒分數、負評比例、常見優點與常見疑慮，並影響外食最終排序。
- 模型解釋器會拆解每一項推薦分數來源，並用敏感度分析比較不同偏好下的排序變化。
- CP 值使用評分與價格換算成 0 到 100 分區間，並用於排序與地圖圖針顏色。
- 內食決策使用集合比對概念，計算使用者現有食材與食譜所需食材的符合數與缺少數。
- 食材保存優先級把保存期限、已放天數、價格與易腐程度轉換成排序分數。
- 推薦結果會產生可解釋理由，讓使用者知道每筆推薦為什麼出現。

## 本機執行方式

### 1. 建立虛擬環境

```bash
python3 -m venv .venv
```

### 2. 啟用虛擬環境

macOS / Linux：

```bash
source .venv/bin/activate
```

```bash
.venv\Scripts\activate
```

### 3. 安裝套件

```bash
pip install -r requirements.txt
```

### 4. 執行網站

```bash
streamlit run app.py
```

開啟瀏覽器後，通常會看到：

```text
http://localhost:8501
```

## 資料集說明

### restaurants.csv

餐廳資料欄位包含：

- `name`：餐廳名稱
- `category`：餐點類型
- `price`：平均價格
- `rating`：評分
- `distance`：距離，單位為步行分鐘
- `serve_speed`：出餐速度
- `takeout`：是否可外帶
- `weather`：適合天氣
- `mood`：適合心情
- `spicy_level`：辣度等級
- `latitude`：緯度
- `longitude`：經度

### recipes.csv

食譜資料欄位包含：

- `name`：食譜名稱
- `category`：食譜類型
- `ingredients`：所需食材
- `missing_allowed`：可接受缺少食材數
- `time`：烹飪時間
- `difficulty`：料理難度
- `calories`：預估熱量
